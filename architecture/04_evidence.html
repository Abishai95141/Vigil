Why this matters.
01 · claim The operator sees a one-line story they can act on. Nothing in the UI hides what's behind it.
02 · reasoning The pattern that fired is named, versioned, and inspectable. There is no opaque model output to argue with.
03 · agent Run trace shows which agent, which inputs, which bindings, and how many LLM tokens. Every cost is accounted for.
04 · raw The bytes that justify the claim live in the customer's own cluster. A skeptical operator can re-run the kubectl query themselves.
Trust contract
Surface, do not act. Every claim is operator-verifiable. No mystery boxes.
